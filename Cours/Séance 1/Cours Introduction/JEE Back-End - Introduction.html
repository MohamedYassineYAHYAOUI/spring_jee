Introduction JEE

Java Entrepise Edition (JEE)

JEE est une plate-forme logiciel qui définit:

  • des spécifications
  • des APIs

... pour écrire des applications Java pour l'entreprise.

  • La version courante de JEE est 1.8.
  • JavaEE = Java SE + un grand nombre de bibliothèques
  • De nombreux frameworks implémentent tout ou partie de ces spécifications.

Pour qui ? Pourquoi ?

  • pour des projets d'envergure qui doivent évoluer sur une longue période (banque, ...)
  • évolutions technologiques inévitables et difficilement anticipables (applications mobiles, big data,...)
  • en contre-partie, un code beaucoup plus compliqué ...

Avantages JEE

  • une architecture d'applications basée sur les composants
    • découpage de l'application
    • séparation des rôles
  • interfaçage possible avec de nombreuses API : JDBC, JPA, JSP...
  • choix des outils de développement et des serveurs d'applications utilisés
  • Découpage de l'application : client/métier/données, MVC

Une vue d'ensemble

  • Front-end : clients
  • Back-end : midleware + BDD
  • Contrôleurs : Point d'entrée/sortie client midleware

Spring

Spring est un framework Java libre qui implémente les spécifications JEE et fournit d'autres modules qui ne sont pas dans la spécification.

  • Rod Johnson (2002)
  • Version 5 actuellement
  • Très modulaire
  • Simple : point de vue unifié
  • Rapide : serveur léger (Attention au passage à l'échelle)
  • Testable
